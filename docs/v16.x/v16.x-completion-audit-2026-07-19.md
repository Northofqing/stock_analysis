## 数据流与接口

```text
真实账户/能力评估
  -> monitor LATEST_BANNER
  -> PaperRiskContext(AccountMode, DataMode)
  -> intraday/evening/paper-exit
  -> order_safety + risk_adapter
  -> paper_trades + immutable order audit

完整持仓行情批次
  -> strict code/price/change validation
  -> decisions_from_llm(Result<Vec<FinalDecision>>)
  -> decision renderer / governed delivery
```

`PaperRiskContext` 是风险事实的唯一传递接口；调用方不能再传字符串，也没有默认实现。
纸面持仓恢复以真实 Filled 行重建，禁止汇总 SQL 用 0 填补缺失。

## 失败模式

| 失败 | 行为 |
| --- | --- |
| banner 尚未完成真实评估或锁损坏 | 不运行纸面决策/退出，保留下一轮重试 |
| 行情批次缺代码、坏价格、坏涨跌幅 | 决策整批 `Err`，不渲染成本价替代卡片 |
| paper Filled 行缺成交价、非整手、未知方向、超卖 | 持仓恢复整批 `Err`，不触发退出单 |
| 单票退出失败 | 继续尝试本批其他退出，末尾汇总 `Err`；5 分钟成功 debounce 不推进，下一轮仍可重试 |
| paper 结果为 NotFilled/Invalidated 或重复行 | 不发布 `ExecutionFilled`；重复行不发布任何新订单事实 |
| 当前买价达到真实涨停价 | PaperSignal 标记 limit-up，由统一成交评估返回 NotFilled |
| 未实现 broker source | 启动显式失败，不切换到公共源或 mock |
