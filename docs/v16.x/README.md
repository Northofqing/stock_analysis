## 当前可执行合同

- v16.3–v16.6 已落地的纸面交易、推送票池、8 策略、绩效、监控能力，必须同时以
  生产调用链、业务规则登记和当前测试证据为准。
- 生产纸面订单只消费真实且新鲜的行情、账户快照、`AccountMode` 和 `DataMode`；
  缺失即拒绝，不允许成本价、0、`Normal` 或 `Full` 代填。
- 当前唯一已实现的生产行情适配器是 `BROKER_SOURCE=public|tencent`。IB/TQS/QMT
  仍是外部集成提案，不得注册 logging-only/mock provider，也不得宣称已接券商。
- 真实账户快照允许账户号、可取现金、仓位比例等非必要字段为 SQL `NULL`；必填的
  会计字段、来源、时点和所有权声明必须完整。详见 BR-103。
- v16.6 的有效端到端证据是当前编译并运行的测试与隔离进程测试；未被模块树引用的
  历史测试草稿不计入覆盖率或验收。
