## 二、修复的关键 Bug (验证过程中发现)

### Bug 1: 测试环境 FEISHU_TO 缺失

**现象**: 第一次跑 16/20 失败
**根因**: `cargo test` 启动时未继承 shell 的 FEISHU_TO 环境变量, 导致 `resolve_feishu_target()` 返回 None, `push_via_magiclaw_cli` 提前返回 false
**修复**: 测试 set_up 显式设 `FEISHU_TO=oc_4bca5d870fd5ff3352795a674194d5b0`, 或加到 .env
**影响**: 之前 cargo test 跑监控的所有 E2E 测试都隐式依赖 shell 环境, 不可移植

### Bug 2: 测试环境 MAGICLAW_HOME 缺失

**现象**: `push_via_magiclaw_cli` 在 test env 找不到 magiclaw 项目根目录 (`.env`)
**根因**: `resolve_magiclaw_home()` 默认从二进制路径推导, 但 test binary 路径与 cargo run 不同
**修复**: 测试 set_up 显式设 `MAGICLAW_HOME=/Users/zhangzhen/Desktop/magiclaw`
**影响**: 测试环境无法用真实飞书

### Bug 3: 测试环境 DATABASE_PATH / MAGICLAW_DB_PATH 缺失

**现象**: magiclaw spawn 成功但 daemon 启动时 DB 路径错误
**根因**: cargo test 默认无 DATABASE_PATH, `push_via_magiclaw_cli` fallback 到 `./data/stock.db` (旧路径)
**修复**: 测试 set_up 显式设 `DATABASE_PATH=./data/stock_analysis.db` + `MAGICLAW_DB_PATH=./data/stock_analysis.db`
**影响**: 与 Bug 2 同源

### Bug 4: 测试环境 FEISHU_RECEIVE_ID_TYPE 缺失

**现象**: magiclaw 用默认 receive_id_type 而非 `chat_id` (FEISHU_TO 是 chat_id)
**根因**: cargo test 无 FEISHU_RECEIVE_ID_TYPE, magiclaw 用 open_id 接收 → 错配
**修复**: 测试 set_up 显式设 `FEISHU_RECEIVE_ID_TYPE=chat_id`

## 三、E2E 测试设计

### 真实数据装配策略

| 数据类型 | 来源 | 失败回退 |
|---|---|---|
| ledger (today_pnl, total_value) | `stock_analysis::portfolio::get_equity_curve(1)` | 0.0 (E2E 仍能跑) |
| open positions | `stock_analysis::portfolio::get_positions()` | 合成 `("示例持仓", "000001", 11.80, 3000)` |
| trades history | `stock_analysis::portfolio::get_trade_history(30)` | 空 |
| prev AccountMode | `account_mode_log::latest_account_mode_change()` | Normal |
| Capability status | 硬编码 5 维全 fresh=200 (触发 Degraded→Unsafe) | - |
| market_stage_confidence | 真 3 维 (技术/资金/情绪) | - |
| limit_chain_review | 真 2 只 AI 算力 (4 板 + 2 板) | - |
| lhb_review | 合成 1 条 (688001 净买 1.5亿) | - |
| failure_review | 合成 1 条 (涨停不可买) | - |

### Opt-in 机制

`V12_E2E_REAL_PUSH=1` env var 控制. 默认 CI 不跑 (避免污染飞书群), 显式 opt-in 才执行真推.
`PUSH_VERBOSE=true` 同时设, 让 deprecated PushKind 也真推 (默认降级 log, E2E 不需要).

### 消息前缀

每条消息带 `[v12-E2E-T0X]` 前缀, 便于:
1. 飞书群识别 (可批量删除测试消息)
2. log 关联 (grep E2E 即得 20 模板结果)
