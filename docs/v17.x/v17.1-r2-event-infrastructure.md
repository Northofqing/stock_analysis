## 0. 一句话定位

v17.1-r2 = **在 v14.2 push 栈 (L4-L7) 上方增加 DomainEvent + EventBus 基础设施层**，让业务事件通过标准化 envelope 流向唯一的 `PushApplicationService`，并彻底修复 5 个隐藏 P0 bug。

**核心原则**：
- **不替代 L4-L7**（reuse 而非 replace）
- **EventBus 零业务**（不写 `PushKind`，不写 `sink_name`）
- **DispatcherRegistry 用 `Box<dyn Dispatcher>`**（不再 `match`）
- **可靠交易事件走 ReliableChannel**（mpsc + ACK，非 broadcast）

**ReliableChannel 关键特性**：
- `try_send` 非阻塞；满返 BackpressureError
- `pending_acks: DashMap<Uuid, Instant>` 追踪未 ACK
- `ack(event_id)` consumer 处理完调
- **ACK 闭环补全 (防泄漏)**：channel 自持后台 sweeper (每 10s 一轮)，将 `elapsed > ack_timeout` (默认 30s，可配) 的条目移除 + warn log + `metrics.ack_timeout_total += 1` (静默路径可见)。超时**只记账不重投** —— 重投交给周期兜底 dispatcher，避免重试风暴。consumer panic / 永不 ack 时 `pending_acks` 不再无限增长
- 容量 1024 默认；可配

### 5.6 L4 原子化（修复并发双发）

> 业务规则登记: **BR-030** (docs/business_rules.md) — (kind, code) 冷却窗 dedup 属 dedup 类逻辑，按红线 2.10 先注册。

```rust
pub struct Dispatcher {
    dedup_table: DashMap<(String, String), Reservation>,
}

pub struct Reservation {
    event_id: Uuid,
    expires_at: Option<Instant>,   // None = 从未 reserve 过, 视为已过期
    committed: bool,
}

pub enum ReserveOutcome { Reserved(Reservation), Deduped }
pub enum CommitResult { Committed, Expired, AlreadyCommitted }

impl Dispatcher {
    pub fn reserve(&self, event: &SignalEvent, cooldown: Option<Duration>) -> ReserveOutcome {
        let key = (event.kind.clone(), event.code.clone().unwrap_or_default());
        let cooldown = cooldown.unwrap_or(DEFAULT_COOLDOWN);
        // 修复 v17.1-r2 #1: DashMap::entry() 返回的 RefMut 在 scope 内独占 shard,
        //   只要把 entry 持到 scope 末尾 + 不依赖 committed 字段,
        //   check-and-modify 真正原子 (修复前 committed=true 才 dedupe,
        //   → A reserve 后未 commit 期间 B 来 reserve 仍能拿到 token,导致双发).
        let mut entry = self.dedup_table.entry(key)
            .or_insert_with(|| Reservation::expired());
        let now = Instant::now();
        if entry.expires_at.is_some_and(|e| e > now) {
            // 时间窗未过期 (不管 committed 与否) → Deduped
            return ReserveOutcome::Deduped;
        }
        entry.event_id = Uuid::new_v4();
        entry.expires_at = Some(now + cooldown);
        entry.committed = false;
        ReserveOutcome::Reserved(Reservation {
            event_id: entry.event_id,
            expires_at: entry.expires_at,
            committed: false,
        })
    }
    
    pub fn commit(&self, key: &(String, String), res: Reservation) -> CommitResult {
        // 验证 event_id 匹配 + 标记 committed
        match self.dedup_table.entry(key.clone()) {
            dashmap::mapref::Entry::Occupied(mut e) if e.get().event_id == res.event_id => {
                e.get_mut().committed = true;
                CommitResult::Committed
            }
            _ => CommitResult::Expired,
        }
    }
    
    pub fn rollback(&self, key: &(String, String), res: Reservation) {
        // 删除占位; 避免"投递失败占满 24h"老 bug
        self.dedup_table.remove_if(key, |_, v| v.event_id == res.event_id);
    }
}

#[derive(Clone, Debug)]
pub struct Reservation {
    pub event_id: Uuid,
    pub expires_at: Option<Instant>,   // None = 已过期 (从未 reserve)
    pub committed: bool,
}

impl Reservation {
    pub fn expired() -> Self {
        Self {
            event_id: Uuid::nil(),
            // 注意: 不可写 Instant::now() - Duration::from_secs(1) —
            //   Instant 减法在启动 <1s 的平台上会 panic (已知 pitfall), 用 None 表达"已过期"
            expires_at: None,
            committed: false,
        }
    }
}
```

#### 5.10.2 SinkRouter（路由 + 重试 + 观测）

```rust
// src/push_l6/sink_router.rs (新增)

pub struct SinkRouter {
    sinks: Vec<Arc<dyn Sink>>,
    strategy: DefaultStrategy,                  // toml 配置默认策略
    retry_policy: RetryPolicy,                  // toml 配置重试
    metrics: Arc<SinkMetrics>,
}

impl SinkRouter {
    pub fn new(sinks: Vec<Arc<dyn Sink>>, strategy: DefaultStrategy) -> Self;

    /// 主入口：按 msg.channel_strategy 路由
    pub async fn route(&self, msg: &OutboundMessage) -> Vec<SinkResult> {
        let strategy = if matches!(msg.channel_strategy, ChannelStrategy::All) {
            self.strategy                              // 未指定 → 用 toml 默认
        } else {
            msg.channel_strategy
        };

        let enabled: Vec<&Arc<dyn Sink>> = self.sinks.iter().filter(|s| s.enabled()).collect();
        if enabled.is_empty() {
            log::error!("[sink_router] NO sink enabled — 消息丢弃 kind={:?}", msg.kind);
            self.metrics.no_sink_total.fetch_add(1, Ordering::Relaxed);
            return vec![];
        }

        match strategy {
            ChannelStrategy::All       => self.send_all(&enabled, msg).await,
            ChannelStrategy::Failover  => self.send_failover(&enabled, msg).await,
            ChannelStrategy::RoundRobin=> self.send_round_robin(&enabled, msg).await,
        }
    }

    async fn send_all(&self, sinks: &[&Arc<dyn Sink>], msg: &OutboundMessage)
        -> Vec<SinkResult>
    {
        let mut results = Vec::with_capacity(sinks.len());
        for sink in sinks {
            let r = self.send_with_retry(sink.as_ref(), msg).await;
            self.metrics.record(&r);                  // 静默路径可见
            results.push(r);
        }
        results
    }

    async fn send_failover(&self, sinks: &[&Arc<dyn Sink>], msg: &OutboundMessage)
        -> Vec<SinkResult>
    {
        for sink in sinks {
            match self.send_with_retry(sink.as_ref(), msg).await {
                ok @ SinkResult::Ok { .. } => {
                    self.metrics.record(&ok);
                    return vec![ok];
                }
                other => {
                    self.metrics.record(&other);     // 记失败，下一个 sink 兜底
                }
            }
        }
        log::error!("[sink_router] all sinks failed kind={:?} trace_id={}",
            msg.kind, msg.trace_id);
        Vec::new()
    }

    async fn send_with_retry(&self, sink: &dyn Sink, msg: &OutboundMessage) -> SinkResult {
        let mut attempt = 0u32;
        loop {
            match sink.send(msg).await {
                ok @ SinkResult::Ok { .. } => return ok,
                fail @ SinkResult::Failed { retryable, .. } if retryable && attempt < self.retry_policy.max_retries => {
                    attempt += 1;
                    self.metrics.retry_total.fetch_add(1, Ordering::Relaxed);
                    log::warn!("[sink_router] retry attempt={} sink={} kind={:?}",
                        attempt, sink.name(), msg.kind);
                    tokio::time::sleep(self.retry_policy.backoff(attempt)).await;
                }
                other => return other,
            }
        }
    }
}

#[derive(Debug, Clone)]
pub struct RetryPolicy {
    pub max_retries: u32,                      // 默认 2
    pub backoff: Backoff,                      // 指数退避，默认 base=500ms, factor=2
}
impl RetryPolicy {
    pub fn backoff(&self, attempt: u32) -> Duration {
        Duration::from_millis(self.backoff.base_ms * 2u64.pow(attempt - 1))
    }
}
```

```rust
        metadata: BTreeMap::new(),

    };
```

### 13.4 运行时语义 (10-12)

| # | 决策 | 选择 | 理由 |
|---|------|------|------|
| 10 | **Backpressure 策略**（ReliableChannel 满 1024） | **`publish()` 立即返 `false` + warn log + metrics.publish_backpressure++** | 业务不被卡；监控可见 |
|    | 上游应对 | 调用方负责：retry / 改走 DropOnFull / log warn | 与 EventBus 解耦 |
|    | 监控 | banner + /metrics 含 `event_bus.publish_backpressure_total` | 满足 v15.x 静默路径可见 |
| 11 | **Shutdown drain 期间 publish 行为**（30s 内） | **`publish()` 立即返 `false` + `reason=shutdown`** | 明确边界；in-flight 事件走完 L4→L6 后清 |
|    | in-flight 处理 | 已 reserve 的 Reservation 走完 L6 或 rollback；不丢弃 | 避免"半推" |
|    | 监控 | banner `event_bus.shutdown_publish_rejected_total` | 排查"为何没推" |
| 12 | **Dispatcher 路由冲突**（多 dispatcher accept 同一事件） | **注册顺序**：Vec<Dispatcher> 顺序遍历，`accepts()` true 即调 + 早退 | 与 std::sync::Mutex 习惯一致；实现最简 |
|    | 排序约束 | 注册代码内注释声明顺序；v17.3 可能改 trie | 不在 r2 引入 trie |
|    | 冲突检测 | 启动时 `DispatcherRegistry::validate()` 警告 `event_type covered by 2+ dispatchers` | 提前发现误配 |
