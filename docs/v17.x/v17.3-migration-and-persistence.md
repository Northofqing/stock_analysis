## 0. 一句话定位

v17.3 = **在 r2 基础设施之上,把业务全部迁入 DomainEvent / Dispatcher 体系**,补齐 v17.1/v17.2 遗留的 6 大块 (PushKind × 30, emit_buy, 4 Bus, L3/L5/L7, JSONL, Replay/History),并彻底整合 4 套旧 Bus。

**核心原则**：
- **复用 r2 基础设施**(DomainEvent trait / EventEnvelope / ChannelRegistry / DispatcherRegistry)
- **迁移而非替换**(PushKind enum 保留,只改 emit 路径走 EventBus)
- **JSONL append-only**(7 天 + 按日轮转,只追加不重写)
- **Replay 默认 dry-run**(防误推,需 `--replay-force`)

**PushKindDispatcher 注册**(D1)：

```rust
// src/event/migration/push_kind_dispatcher.rs (修复 v17.3 #9-r3: MIGRATED_KINDS 校验 fail-fast, 弃用 58 全集校验 + unsafe transmute)

pub fn register_all(registry: &mut DispatcherRegistry, service: Arc<PushApplicationService>) {
    let mut registered: HashSet<PushKind> = HashSet::new();
    
    // 批 A: 业务信号(顺序敏感,先注册 → 路由优先)
    for kind in BUSINESS_KINDS {  // 12 个
        registry.register(Arc::new(PushKindDispatcher::new(kind, service.clone())));
        registered.insert(kind);
    }
    // 批 B: 通道信号
    for kind in CHANNEL_KINDS {  // 9 个
        registry.register(Arc::new(PushKindDispatcher::new(kind, service.clone())));
        registered.insert(kind);
    }
    // 批 C: 状态变更
    for kind in STATE_KINDS {  // 9 个
        registry.register(Arc::new(PushKindDispatcher::new(kind, service.clone())));
        registered.insert(kind);
    }
    
    // 修复 #9-r3: 启动期校验只针对"本期计划迁移的 30 个"(MIGRATED_KINDS 常量表)
    //   ⚠️ 不做 58 全集校验 — 只迁 30 个却校验 58 个会每次启动固定打 28 条 error,
    //     把真告警淹没(狼来了); 未迁移的 28 个走 legacy push_governor 是设计内状态
    //   ⚠️ 禁用 unsafe transmute 遍历(假设 repr(u32) + 判别值连续, 变体增删即 UB) —
    //     PushKind 加 #[derive(strum::EnumIter)], 用 PushKind::iter() 安全遍历
    let missing: Vec<PushKind> = MIGRATED_KINDS.iter()
        .filter(|k| !registered.contains(k))
        .copied()
        .collect();
    if !missing.is_empty() {
        // 漏注册 = 代码 bug → fail-fast panic (配置错误不带病上线, 而非 error log 自欺)
        panic!("[push_kind_dispatcher] MIGRATED_KINDS 中 {} 种未注册: {:?}", missing.len(), missing);
    }
    // 未迁移变体可见性 (v15.x 静默路径可见): 启动 banner 打一行 info
    let total = PushKind::iter().count();
    log::info!("[push_kind_dispatcher] migration: {}/{} migrated, {} legacy via push_governor",
        MIGRATED_KINDS.len(), total, total - MIGRATED_KINDS.len());
    
    // 启动时 validate (r2 决策 #12 锁定)
    registry.validate().expect("config error: dispatcher conflict");
}
```

**PushKindDispatcher 单实现**(D4 模板函数)：

```rust
pub struct PushKindDispatcher {
    pub kind: PushKind,
    pub service: Arc<PushApplicationService>,
}

#[async_trait]
impl Dispatcher for PushKindDispatcher {
    fn name(&self) -> &'static str { push_kind_to_event_type(self.kind) }
    
    fn accepts(&self, env: &EventEnvelope<...>) -> bool {
        env.event_type == self.name()  // 精确匹配,不用 starts_with (r2 #12 锁定)
    }
    
    async fn dispatch(&self, env: EventEnvelope<...>) -> DispatchResult {
        let pe = env.payload.downcast::<PushKindEvent>()
            .ok_or_else(|| Failed("type mismatch".into()))?;
        
        // L3 render (D4)
        let text = l3_render::render(self.kind, pe.as_ref());
        
        // PushApplicationService::push (唯一入口)
        let outcome = self.service.push(self.kind, pe.code.as_deref(), text, pe.trace_id).await;
        
        match outcome {
            PushOutcome::Pushed => Handled,
            PushOutcome::Deduped => Skipped(format!("dedup: kind={:?}", self.kind)),
            PushOutcome::Denied(why) => Skipped(format!("denied: {}", why)),
            PushOutcome::Failed(e) => Failed(e.to_string()),
        }
    }
}
```

### 5.3 4 套旧 Bus 整合(D3: legacy_bridge)

**整合策略**：**legacy_bridge**(impl EventBus for 旧 Bus,主路径仍走 r2 EventBus)

| 旧 Bus | 现状 | v17.3 整合方式 |
|--------|------|----------------|
| `Monitor EventBus` (broadcast 256, 2 producer) | 1 个写日志 consumer | **impl r2 EventBus via legacy_bridge**;内部 mpsc+broadcast 兼容 |
| `v16.4 SignalBus` (broadcast 512, 0 producer) | 0 consumer | **deprecated wrapper**,publish() 返 false + 首次调用 warn log `legacy_bus:signal:deprecated_use_main_event_bus` |
| `v16.4 TradingBus` (broadcast 128, 2 producer) | 0 consumer | **deprecated wrapper**,publish() 返 false + 首次调用 warn log `legacy_bus:trading:deprecated_use_main_event_bus` |
| `v16.4 SystemBus` (broadcast 64, 0 producer) | 0 consumer | **deprecated wrapper**,publish() 返 false + 首次调用 warn log `legacy_bus:system:deprecated_use_main_event_bus` |

```rust
// src/event/migration/legacy_bus_bridge.rs

pub struct LegacyBusBridge {
    monitor_bus: Arc<monitor::event_bus::MonitorEventBus>,
    signal_channel: Arc<BroadcastChannel<SignalEvent>>,
    trading_channel: Arc<ReliableChannel<TradingEvent>>,
    system_channel: Arc<DropOnFullChannel<SystemEvent>>,
}

impl EventBus for LegacyBusBridge {
    fn publish(&self, env: EventEnvelope<...>) -> bool {
        // 注意: match arm 写 "signal.*" 只会匹配字面字符串 "signal.*", 不是通配 —
        //   必须按 event_type 首段路由:
        match env.event_type.split('.').next() {
            Some("monitor") => self.monitor_bus.publish(env),
            Some("signal") => self.signal_channel.try_send(env).is_ok(),
            Some("trading") => self.trading_channel.try_send_with_ack(env).is_ok(),
            Some("system") => self.system_channel.try_send_drop_on_full(env),
            _ => false,
        }
    }
    
    fn subscribe(&self) -> Box<dyn EventReceiver> { ... }
    fn metrics(&self) -> EventBusMetrics { ... }
    fn shutdown(&self, timeout: Duration) -> ShutdownReport { ... }
}
```

**主路径仍走 r2 EventBus**: `event_bus::global()` 默认返回 r2 EventBusImpl,legacy_bridge 仅作为兼容层(供旧模块未迁移前)。

```rust
pub fn render(kind: PushKind, event: &PushKindEvent) -> String {
```

### 5.5 L5 daily_limit 分类限速(D5)

> 业务规则登记: **BR-031** (docs/business_rules.md) — limit 类逻辑, 按红线 2.10 先注册。

```rust
// src/event/l5_limit.rs (修复 v17.3 #10 + #11 + #12-r3: 真原子 check+increment)

use chrono::Local;
use arc_swap::ArcSwap;

pub struct DailyLimitPolicy {
    pub global_daily: u32,                  // 默认 200
    pub per_kind_daily: HashMap<PushKind, u32>,  // KBuy/KSell/KStopLoss 各 20/天
}

impl Default for DailyLimitPolicy {
    fn default() -> Self {
        let mut per_kind = HashMap::new();
        per_kind.insert(PushKind::KBuy, 20);
        per_kind.insert(PushKind::KSell, 20);
        per_kind.insert(PushKind::KStopLoss, 30);  // 止损多放一点
        Self { global_daily: 200, per_kind_daily: per_kind }
    }
}

/// 修复 #12-r3: 弃用 ArcSwap<HashMap> read-copy-update 方案 —
///   两个并发推送各自 clone 同一基线再 store 会互相覆盖 → **计数丢失**;
///   且 check 与 increment 分两步 (TOCTOU), 限速可被并发穿透。
/// 改为 DashMap<PushKind, AtomicU64> + fetch_add 单步原子 check+increment。
pub struct DailyLimiter {
    /// 修复 #10: day_key = `Local::now().format("%Y-%m-%d")` (本地时区日历日)
    ///   跨天检测: day_key 不匹配 → write lock 内整体 clear + 换 key (每天仅一次),
    ///   顺带解决"日桶 key 永不清理、内存无限涨"问题
    day_key: RwLock<String>,
    kind_counts: DashMap<PushKind, AtomicU64>,
    global_count: AtomicU64,
}

impl DailyLimiter {
    pub fn check_and_increment(&self, policy: &DailyLimitPolicy, kind: PushKind) -> DailyLimitVerdict {
        self.rollover_if_new_day();   // day_key 变化 → clear 所有桶

        // 1. kind 二级桶: fetch_add 后判限, 超限则回退 — 单步原子, 无 TOCTOU
        if let Some(&limit) = policy.per_kind_daily.get(&kind) {
            let c = self.kind_counts.entry(kind).or_insert_with(|| AtomicU64::new(0));
            let prev = c.fetch_add(1, Ordering::Relaxed);
            if prev >= limit as u64 {
                c.fetch_sub(1, Ordering::Relaxed);   // 回退, 计数不虚高
                return Denied(format!("per_kind_limit: {:?} {}/{}", kind, prev, limit));
            }
        }

        // 2. 全局桶: 同样 fetch_add + 超限回退 (含回退 kind 桶, 保持两桶一致)
        let prev_global = self.global_count.fetch_add(1, Ordering::Relaxed);
        if prev_global >= policy.global_daily as u64 {
            self.global_count.fetch_sub(1, Ordering::Relaxed);
            if policy.per_kind_daily.contains_key(&kind) {
                if let Some(c) = self.kind_counts.get(&kind) {
                    c.fetch_sub(1, Ordering::Relaxed);
                }
            }
            return Denied(format!("global_limit: {}/{}", prev_global, policy.global_daily));
        }

        Allowed
    }

    fn rollover_if_new_day(&self) {
        let today = Local::now().format("%Y-%m-%d").to_string();
        if *self.day_key.read().unwrap() != today {
            let mut key = self.day_key.write().unwrap();
            if *key != today {                     // double-check, 只有一个线程执行 reset
                self.kind_counts.clear();
                self.global_count.store(0, Ordering::Relaxed);
                *key = today;
                log::info!("[daily_limit] day rollover → {} (buckets reset)", today);
            }
        }
    }
}
```

```rust
pub fn load_from_toml(toml_path: &Path) -> DailyLimitPolicy {
    let raw = std::fs::read_to_string(toml_path).unwrap_or_default();
    let parsed: toml::Value = toml::from_str(&raw).unwrap_or(toml::Value::Table(Default::default()));
    let default = DailyLimitPolicy::default();
    
    let global = parsed.get("global_daily").and_then(|v| v.as_integer()).map(|v| v as u32).unwrap_or(default.global_daily);
    let mut per_kind = default.per_kind_daily.clone();
    if let Some(table) = parsed.get("per_kind_daily").and_then(|v| v.as_table()) {
        for (k, v) in table {
            if let Some(n) = v.as_integer() {
                if let Ok(kind) = k.parse::<PushKind>() {
                    per_kind.insert(kind, n as u32);
                }
            }
        }
    }
    let policy = DailyLimitPolicy { global_daily: global, per_kind_daily: per_kind };
    log::info!("[daily_limit] loaded: global={} per_kind={:?}", policy.global_daily, policy.per_kind_daily);
```

```rust
// src/event/jsonl_writer.rs (修复 v17.3 #5 + #6-r3)

pub struct JsonlWriter {
    base_dir: PathBuf,           // 默认 "data/event_bus"
    retention_days: u32,         // 默认 7 (D7 锁定)
    receiver: broadcast::Receiver<EventEnvelope<Box<dyn DomainEventErased>>>,
    file_writer: tokio::fs::File,
    current_date: NaiveDate,
    // 修复 #6-r3: 彻底移除 REPLAY_IN_PROGRESS 全局 AtomicBool —
    //   r2 废弃 v17.1 的理由之一就是 "JsonlWriter 与 Replay 用 AtomicBool 耦合(脆弱)";
    //   且全局开关会让 replay 期间到达的**真实事件**也不落盘 → 审计断档(违反红线 2.7)。
    //   改用 envelope.replay_of 逐事件标记 (见 r2 §5.1 EventEnvelope), 无全局状态。
}

impl JsonlWriter {
    /// 修复 #5: 不再调 bus.subscribe_reliable_broadcast() (该方法在 r2 EventBus trait 中不存在)
    ///   改为接 broadcast::Receiver<EventEnvelope> 参数,由 caller 从 r2 ChannelRegistry.subscribe_broadcast() 取得
    pub fn spawn(
        rx: broadcast::Receiver<EventEnvelope<Box<dyn DomainEventErased>>>,
        base_dir: PathBuf,
        retention_days: u32,
    ) -> JoinHandle<()> {
        tokio::spawn(async move {
            Self {
                base_dir, retention_days,
                receiver: rx,
                file_writer: open_writer(&base_dir, today()).await,
                current_date: today(),
            }.run().await;
        })
    }

    async fn run(mut self) {
        self.cleanup_expired().await;
        while let Ok(env) = self.receiver.recv().await {
            // 逐事件判断: 只跳过回放事件本身; 同期到达的真实事件照常落盘 (红线 2.7)
            if env.replay_of.is_some() {
                log::debug!("[jsonl_writer] skip replayed event: {:?}", env.event_type);
                continue;
            }
            self.write_envelope(&env).await.ok();
        }
    }
}
```

```rust
// src/event/replay.rs

#[derive(Args)]
pub struct ReplayArgs {
    pub date: NaiveDate,
    #[arg(long)] pub force: bool,            // --replay-force 真正推
    #[arg(long, default_value = "100")] pub rate: u32,
}

pub async fn run(args: ReplayArgs) -> Result<usize, ReplayError> {
    if !args.force {
        log::warn!("[replay] DRY-RUN mode. Use --replay-force to actually push.");
    }
    
    let path = PathBuf::from("data/event_bus").join(format!("{}.jsonl", args.date));
    let content = tokio::fs::read_to_string(&path).await?;
    let events: Vec<EventEnvelope<...>> = content.lines()
        .filter_map(|l| serde_json::from_str(l).ok())
        .collect();
    
    log::info!("[replay] {} 含 {} 条事件, mode={}", path.display(), events.len(),
        if args.force { "FORCE" } else { "DRY-RUN" });
    
    for mut env in events {
        env.replay_of = Some(env.id);   // 标记回放: JsonlWriter 跳过落盘, l3_render 加 [REPLAY] 前缀
        env.id = Uuid::new_v4();        // 换新 id, 不与原事件的 L4 dedup / ACK 记录冲突
        if args.force {
            event_bus::publish(env.clone());
        } else {
            log::info!("[replay:DRY] would publish: type={} code={:?}",
                env.event_type, env.payload.entity_key());
        }
        tokio::time::sleep(Duration::from_millis(args.rate as u64)).await;
    }
    Ok(events.len())
}
```

**D8 安全锁**：
- 默认 `--replay-dry-run`(只 log,不真推)
- 加 `--replay-force` 才真推
- force 推送文案强制带 `[REPLAY YYYY-MM-DD]` 前缀 (l3_render 检 `env.replay_of`) — 接收端永远能区分回放 vs 实时 (红线 2.1/2.4: 陈旧数据不得伪装成实时推送)

### 5.9 History Query(D9)

```rust
// src/event/history.rs (修复 v17.3 #7: 改 tokio::fs, 不阻塞 executor)

#[derive(Args)]
pub struct HistoryArgs {
    pub date: Option<NaiveDate>,
    #[arg(long)] pub code: Option<String>,
    #[arg(long)] pub kind: Option<String>,
    #[arg(long, default_value = "100")] pub limit: u32,
    #[arg(long, default_value = "DESC")] pub order: HistoryOrder,
}

/// 修复 #7: async fn + tokio::fs::read_to_string
/// (修复前 std::fs::read_to_string 在 tokio runtime 内会阻塞 executor worker,
///  大 JSONL >50MB 时整个 monitor 推送延迟,与 r2 #10 立即失败原则冲突)
pub async fn query(args: &HistoryArgs) -> Result<Vec<HistoryEntry>, HistoryError> {
    let date = args.date
        .or_else(|| Local::now().date_naive().into())
        .ok_or(HistoryError::NoDate)?;
    let path = PathBuf::from("data/event_bus")
        .join(format!("{}.jsonl", date.format("%Y-%m-%d")));
    
    // tokio::fs 异步读,不阻塞 executor
    let content = tokio::fs::read_to_string(&path).await
        .map_err(|e| HistoryError::FileNotFound(path.display().to_string(), e.to_string()))?;
    
    let mut entries: Vec<HistoryEntry> = content.lines()
        .filter_map(|l| parse_line(l))
        .filter(|e| args.code.as_ref().map_or(true, |c| e.code.as_deref() == Some(c.as_str())))
        .filter(|e| args.kind.as_ref().map_or(true, |k| e.kind == *k.as_str()))
        .collect();
    entries.sort_by(|a, b| b.ts.cmp(&a.ts));
    entries.truncate(if args.limit == 0 { usize::MAX } else { args.limit as usize });
    Ok(entries)
}

/// CLI 入口 (async)
pub async fn print_history(args: HistoryArgs) -> Result<usize, HistoryError> {
    let entries = query(&args).await?;
    for e in &entries {
        println!("[{}] {} {} {}", e.ts.format("%H:%M:%S"), e.kind, e.code.as_deref().unwrap_or("-"), e.summary);
    }
    Ok(entries.len())
}
```

**D9 默认值**: limit=100 / order=DESC;`limit=0` 表无限。
