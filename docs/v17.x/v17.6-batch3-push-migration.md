#### 5.1.3 L4 Dedup for DailyReport

```rust
// src/push_l4/dedup.rs (MOD, Fix #11: 扩展 event_id schema 加 sub_kind)
//
// 问题: 现有 push_l4/dispatcher.rs:91-94 dedup key 是 (event.kind, event.code);
// DailyReport 9 callsite 的 code 是 None → dedup key 变 (DailyReportUpdate, "") 全冲突 → 第一子段赢,
// 后续 IntradaySummary / EodSummary 等静默丢。
//
// 修复: 在 DomainEvent 加可选 sub_kind field, dedup 函数读 sub_kind 加入 key。
// 注意: push_l1/event.rs 当前没有 sub_kind field, 这是 v17.6 必做的 schema 扩展。

pub struct DomainEvent {
    pub kind: EventKind,
    pub code: Option<String>,
    pub ts: SystemTime,
    pub sub_kind: Option<DailyReportSubKind>,    // NEW (v17.6)
    pub content: String,
}

fn make_dedup_key(evt: &DomainEvent) -> DedupKey {
    match &evt.kind {
        // DailyReportUpdate 用 (sub_kind, date) 当 key, 同日同子段不重推
        EventKind::DailyReportUpdate => {
            let sub = evt.sub_kind.expect("DailyReportUpdate must have sub_kind");
            let date = NaiveDate::from_ymd_opt(/* evt.content.bucket_date */ 1970, 1, 1).unwrap();
            DedupKey::Hash(hash_of(&(sub, date)))
        }
        // 其他 EventKind 走默认 (kind, code) 路径
        _ => DedupKey::Compound(evt.kind.clone(), evt.code.clone().unwrap_or_default()),
    }
}
```

**降噪效果**: 当前 9 callsite 同一天可能重复推 (L4 没区分),v17.6 后真正 "一日一推"。

### 5.2 EventCalendarUpdate (~80 行, 5 callsite 整合)

```rust
// src/event_calendar_bridge.rs
pub struct EventCalendarBridge {
    last_emit: HashMap<(NaiveDate, EventKind), Instant>,
}

impl EventCalendarBridge {
    /// 5 callsite 路由到统一 emit
    pub fn emit(&mut self, ctx: &str, date: NaiveDate, content: String) -> DomainEvent {
        let key = (date, route_event_kind(ctx));
        if let Some(last) = self.last_emit.get(&key) {
            if last.elapsed() < Duration::from_secs(3600) {
                log::info!("[EventCalendar] 同日期同类型 1h 内重复, 跳过");
                return DomainEvent::noop();   // L1 EventBus 走 noop 路径
            }
        }
        self.last_emit.insert(key, Instant::now());
        DomainEvent::event_calendar_update(date, content)
    }
}
```
