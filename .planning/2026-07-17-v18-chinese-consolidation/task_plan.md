# v18.x 中文整合文档计划

## 目标

将 `docs/v18.x` 现有评估、架构设计、四模块详细设计和实施路线图去重整合为一份中文权威主文档，保留来源追踪与仓库安全门禁。

## 阶段

- [x] 阶段 1：完成仓库规则、工作树和 v18.x 文档清单的只读预检
- [x] 阶段 2：提取四份来源文档的事实、设计决策、验收项与重复点
- [x] 阶段 3：编写中文整合主文档并更新 v18.x 索引
- [x] 阶段 4：检查结构、术语、来源映射、链接与差异
- [ ] 阶段 5：运行仓库规定验证并记录结果（被既有 Rust 格式差异阻断）

## 约束与决定

- 仅修改 `docs/v18.x`、必要的文档索引和本任务 `.planning` 记录。
- 不改变代码、配置阈值、业务规则或生产数据路径。
- 三份伴随材料保留为来源文档；原总体架构在同一已跟踪文件中升级为中文整合入口，旧版可由 Git 恢复。
- 不把来源文档中的完成声明升级为事实；缺少 Gate 证据的内容保持“规划中/待验证”。
- 中文整合内容写入已跟踪的活跃总体设计文件，确保可进入 PR。

## 错误记录

| 错误 | 次数 | 处理 |
| --- | ---: | --- |
| `.github/copilot-instructions.md` 不存在 | 1 | 记录缺失，采用 AGENTS.md、ENGINEERING_RULES_V2.md、CLAUDE.md |

| `cargo fmt --check` 退出 1 | 1 | 差异均在本任务外的既有 Rust 改动；按 Gate 顺序停止，不擅自格式化用户代码 |
